# PPAI 2026 - Grupo 12

¡Bienvenidos al repositorio del PPAI! 

Esta guía es para ayudar a todos los integrantes a contribuir al proyecto de forma fácil y ordenada, especialmente si es tu primera vez usando Git.

## ¿Cómo contribuir al proyecto?

Nuestra rama principal (`main`) está **protegida**. Esto significa que no se pueden subir (pushear) cambios directamente a ella. Para agregar tus cambios al proyecto, debes seguir estos pasos:

### 1. Crear una rama (Branch)
Cada vez que vayas a trabajar en algo (una corrección, agregar un diagrama, etc.), debes crear una rama nueva.

**Formato para el nombre de la rama:**
El nombre debe seguir la estructura `nombre-integrante/proposito`. 

*Ejemplos:*
- `mar/correccion-interaccion`
- `syd/v2-estado`
- `gonza/fix-dominio`

### 2. Subir tus cambios
Trabaja tranquilamente en tu rama, guarda tus cambios y súbelos al repositorio.

### 3. Crear un Pull Request (PR)
Una vez que tus cambios estén listos y subidos, debes ir a GitHub y crear un **Pull Request**. Un PR es básicamente una solicitud para unir el trabajo de tu rama hacia la rama principal (`main`).

- Al crear el PR, aparecerá automáticamente una **plantilla**.
- Completa la información que pide la plantilla describiendo qué cambios hiciste. Esto ayuda a que el resto del equipo entienda fácilmente tu trabajo.

### 4. Revisión y Aprobación
Para asegurarnos de que todo funcione correctamente, es necesario que otro integrante del grupo revise los cambios.

- En la página de tu Pull Request, elige a cualquier compañero del equipo como **Reviewer** (Revisor).
- Tu compañero mirará los cambios y los aprobará.
- **Importante:** La opción para hacer "Merge" (unir tu rama a `main`) solo se va a habilitar después de que alguien haya revisado y **aprobado** tu PR.

¡Y eso es todo! Una vez aprobado y unido (merged), tus cambios formarán parte oficial de la rama `main`. 

Si te trabas en algún paso, ¡pedí ayuda en el grupo!
